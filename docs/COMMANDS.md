# Commands

本文档整理 `daily-reminder_VPS` 的用户命令协议和运行时行为。

## 触发规则

只有消息明确以 `每日提醒 + 动词` 开头时，才进入该 skill。

支持的动词：

- `开始`
- `新增`
- `第N条完成`
- `第N条改到 HH:mm`
- `查看`
- `停止`

## 命令参考

### `每日提醒 开始`

用途：

- 启动当天第一批任务
- 如果当天已经存在一套提醒，则按追加任务处理

示例：

```text
每日提醒 开始
今天先写日报，下午三点半提醒我跟进客户，晚上把方案再过一遍
```

结果：

- 任务写入当天状态
- 新任务按顺序分配编号
- scheduler 继续按整点、半点和专项时间提醒

### `每日提醒 新增`

用途：

- 给当天已有任务继续追加内容

示例：

```text
每日提醒 新增
再加两条，一个是给客户回消息，一个是六点提醒我交电费
```

结果：

- 编号继续递增
- 已完成任务编号不变
- 如果当天此前已经全部完成，则会恢复提醒

### `每日提醒 第N条完成`

用途：

- 将某一条任务标记为完成

示例：

```text
每日提醒 第3条完成
```

结果：

- 该条任务保留，但显示删除线
- 编号不重排
- 如果这是最后一条未完成任务，会进入 `paused_all_done`

### `每日提醒 第N条改到 HH:mm`

用途：

- 修改单条任务的专项提醒时间

示例：

```text
每日提醒 第2条改到 15:30
```

结果：

- 只修改专项时间
- 编号和内容不变
- 如果改到过去时间，系统应先追问，不直接落库

### `每日提醒 查看`

用途：

- 随时查看当天完整清单

示例：

```text
每日提醒 查看
```

结果：

- 返回完整任务列表
- 显示已完成和未完成状态

### `每日提醒 停止`

用途：

- 提前结束当天提醒

示例：

```text
每日提醒 停止
```

结果：

- 清空当天状态
- 停止后续提醒

## 解析规则

- 任务内容允许口语化输入
- 如果系统无法确定该拆成几条任务，必须先确认
- 如果时间表达模糊，必须先追问成具体时间
- 如果新增内容与已有任务高度相似，先确认是否重复新增

## 提醒规则

### 常规提醒

- 固定在整点和半点触发
- 每次发送完整清单

### 专项提醒

- 每条任务最多一个时间点
- 到点时也发送完整清单
- 若与整点或半点撞上，则合并为一条

### 完成态

- 完成不是删除
- 后续提醒继续保留该条任务，并显示删除线

### 跨天

- `00:00` 清空状态
- 不保留前一天日志
- 不发送额外结束消息
